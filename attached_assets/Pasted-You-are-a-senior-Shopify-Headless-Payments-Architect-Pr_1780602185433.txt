You are a senior Shopify Headless + Payments Architect.

Project Context:

* Shopify is being used ONLY as the backend (products, inventory, orders, customers).
* Frontend is a custom React storefront hosted on Replit.
* APIs in use:

  * Shopify Storefront API
  * Shopify Customer Account API
* Existing payment methods:

  * Cash on Delivery (COD)
  * InstaPay
* New payment gateway:

  * Paymob (Egypt)
* Goal:

  * Add Credit Card and Debit Card payments using Paymob directly.
  * Do NOT use Shopify Payments.
  * Do NOT use Shopify Checkout.
  * Do NOT use Shopify Payment Transactions.
  * Do NOT attempt to capture, authorize, or record payments through Shopify.
  * Eliminate the "Order has no Shopify payment" error by removing all Shopify payment transaction dependencies.

Paymob Documentation:

* https://developers.paymob.com/paymob-docs/getting-started/overview
* https://developers.paymob.com/paymob-docs/developers/authentication-request-generate-auth-token-1

Required Architecture:

1. Customer selects Credit/Debit Card during checkout.

2. React frontend calls backend endpoint:
   POST /api/paymob/create-payment

3. Backend:

   * Generates Paymob auth token.
   * Creates Paymob order.
   * Creates Paymob payment key/intention.
   * Returns payment session information to frontend.

4. Frontend redirects customer to Paymob hosted payment page OR embedded payment flow (whichever Paymob recommends).

5. Customer completes payment on Paymob.

6. Paymob webhook verifies payment status.

7. Only after successful payment:

   * Create a Shopify order using Storefront API/Admin API.
   * Mark payment status as paid through order metadata only.
   * Do not create Shopify payment transactions.
   * Do not call any Shopify payment APIs.
   * Do not attempt captures or authorizations in Shopify.

8. Save the following mapping in database:

   * Shopify Order ID
   * Shopify Order Number
   * Paymob Transaction ID
   * Paymob Order ID
   * Payment Method
   * Amount
   * Currency
   * Payment Status
   * Date Created
   * Customer Email

9. Add a Shopify order metafield or note containing:

   Paymob Transaction ID: XXXXX
   Payment Method: Card
   Paymob Order ID: XXXXX
   Payment Status: Paid

10. Orders should appear normally inside Shopify Admin for fulfillment and manual approval.

Critical Requirements:

* Remove any logic that calls:

  * Shopify payment transaction APIs
  * recordShopifyPaymentTransaction
  * captureShopifyPayment
  * authorizeShopifyPayment
  * completeShopifyDraftOrder payment flows

* Search entire codebase for any payment ownership assumptions that require a Shopify payment object.

* Ensure card payments work even if Shopify has zero knowledge of the actual payment transaction.

* Payment source of truth must be Paymob.

* Shopify should act only as:

  * Product database
  * Inventory database
  * Customer database
  * Order management system
