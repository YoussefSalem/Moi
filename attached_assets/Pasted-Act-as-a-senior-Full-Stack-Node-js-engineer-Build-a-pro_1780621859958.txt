## Architecture

The frontend is a custom Hydrogen storefront.

Shopify Checkout must NOT be used.

Customers complete checkout on my storefront, pay via Paymob, and orders are created through the Shopify Admin API only after payment status is confirmed.

## API Routes

### POST /api/checkout

Accept:

* cart items
* customer details
* shipping address
* billing address
* total amount

Process:

1. Create a Paymob Intention.
2. Obtain the client_secret.
3. Submit card payment using Paymob APIs.
4. Handle 3DS V2 flows.
5. Return:

   * payment status
   * redirect_url if 3DS is required
   * transaction reference

Store a temporary pending checkout record before returning.

### POST /api/webhooks/paymob

Requirements:

1. Verify Paymob webhook HMAC signature.
2. Reject invalid signatures.
3. Parse:

   * success
   * transaction_id
   * order reference
   * amount
4. Respond immediately with HTTP 200.
5. Process webhook logic asynchronously.

Implement idempotency protection so duplicate Paymob webhooks never create duplicate Shopify orders.

## Shopify Order Creation

If payment success === true:

Create a Shopify Order using Admin API.

Requirements:

* financial_status = paid
* include customer details
* include line items
* include shipping address
* store Paymob transaction_id
* store payment metadata
* store gateway name
* add tags:

  * paymob
  * paid

If payment success === false:

Create a Shopify Draft Order instead.

Requirements:

* preserve cart contents
* preserve customer information
* tag as payment_failed
* include Paymob transaction details
