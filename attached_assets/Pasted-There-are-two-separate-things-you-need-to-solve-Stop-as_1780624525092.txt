There are two separate things you need to solve:

Stop asking Shopify for a field that doesn’t exist (shopify_payment)
Define a clean sync pattern between Paymob and Shopify that doesn’t rely on that field
I’ll walk through both.

1. Stop referencing shopify_payment on the Shopify side

In the Shopify Admin REST API 2024-04, the Order resource simply does not have any shopify_payment field. That’s why you’re seeing an error like:

Order has no shopify_payment
That error is not coming from Shopify itself; it’s coming from one of:

Your backend model / mapping code (e.g., a serializer or ORM expecting an order.shopify_payment property), or
A third‑party integration / library that was written with the wrong assumption.
Concrete fix:

Anywhere in your backend code that reads from or writes to shopify_payment on an order object, remove it or replace it with one of:
Your Paymob transaction fields (from Paymob API)
Or a Shopify metafield / note_attributes you manage yourself
Because your actual Shopify calls are:

Remove that property from your order model, DTO, or serializer.
Replace it with something you control (e.g. a paymob_transaction_id field you populate yourself – see next section).
