Build a Judge.me-style email review system for a Shopify-like store where the email behaves like a structured review form that submits data into the website review system via prefilled redirect.

Core Inspiration:

* Match the UX flow used by Judge.me
* The email should feel like a guided, frictionless review funnel, not just a link.

⸻

1. Email Review Form UI

Design the email as a compact “review card” containing:

A. Mood-based rating selector (primary input)
Replace stars with emoji buttons:

😡 😕 😐 🙂 😍

Requirements:

* Emojis must appear monochrome / desaturated (no color styling) while keeping their natural shape and expression.
* No colored background or gradients behind emojis.
* Each emoji is a single clickable selection (like a button).
* Large tap targets for mobile.

B. Optional Review Input

* Short text field: “Write your feedback”
* Optional name field (if needed)

C. Primary CTA

* Button: “Send Review”

⸻

2. Submission Behavior (Important)

Email clients cannot submit real forms, so:

* Each interaction updates a generated URL state.
* Selecting an emoji sets the rating value.
* Typing feedback is stored via query encoding (or step-based redirect flow if needed).
* The “Send Review” button redirects to the website with all data prefilled.

Example URL structure:

* rating (1–5 mapped from emoji)
* text
* optional name
* order_id / customer_id

Example:
/review?rating=5&text=Great+service&order_id=123

⸻

3. Website Integration

On redirect:

* Auto-fill the review form on the website.
* Pre-populate:
    * emoji rating → numeric score
    * review text
    * customer/order identifiers
* User lands directly on a ready-to-submit review page.
* Final submit creates the review in the database.

⸻

4. UX Requirements

* Must feel like a premium guided review flow (Judge.me-style).
* Clear sections:
    * “How was your experience?”
    * emoji selector row
    * optional feedback box
    * send button
* Minimal friction, one-screen email layout.
* Mobile-first design.

⸻

5. Dark Mode Support

* Ensure excellent contrast in dark mode email clients.
* No pure black-on-black or washed-out gray issues.
* Emoji must remain clearly visible in both light and dark modes.
* Use dark-mode-safe:
    * card backgrounds
    * borders
    * CTA button
    * text
* Maintain WCAG-compliant contrast ratios across Gmail, Apple Mail, Outlook.

⸻

6. Email Compatibility Constraints

* Must work across Gmail, Outlook, Apple Mail, mobile apps.
* No real <form> submission reliance.
* Use only clickable links / buttons.
* Ensure fallback behavior if CSS is stripped.

⸻

7. Conversion Optimization

* Reduce steps to absolute minimum.
* One decision → one action → send.
* Make emoji selection feel instant and satisfying.
* Keep CTA always visible and dominant.
* Prioritize mobile usability above desktop.

⸻

Outcome:
A Judge.me-like embedded email review funnel where users:

1. Tap an emoji (mood rating)
2. Optionally write feedback
3. Click send
4. Land on a prefilled review page ready for final submission
