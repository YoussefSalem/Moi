Excellent detective work pulling up those logs! You have successfully found the exact reasons your MOI storefront is currently showing a white screen.

There are three separate things happening here, but they all stem from Hydrogen's strict security rules and folder structure.

Shopify Oxygen uses something called a Content Security Policy (CSP). Think of it as a very strict bouncer at the door of your website. By default, it blocks everything from loading unless you explicitly put it on the VIP guest list. Right now, the bouncer is blocking your Google Fonts, and more importantly, it is blocking the React code needed to draw your website.

Here is exactly how to fix all three issues in Replit:

1. Fix the White Screen (The Blocked Scripts)
The React code that actually makes your website visible is being blocked because the bouncer doesn't recognize it. In Hydrogen, every script must carry a "VIP pass" called a nonce.

In your Replit file tree, open app/root.tsx.

Look through the file for any <script> or <style> tags you might have added.

You need to make sure every single one includes the nonce property. It should look exactly like this:

TypeScript
<script nonce={nonce}>
  // your script code here
</script>
Also, make sure the main Hydrogen <Scripts /> component at the bottom of your <body> has it: <Scripts nonce={nonce} />.

2. Whitelist your Custom Typography (Google Fonts)
The bouncer is currently rejecting your Cormorant Garamond and Montserrat fonts because they come from an outside URL. We need to add Google to the VIP list.

Open app/entry.server.tsx.

Look for the createContentSecurityPolicy function we looked at earlier.

You need to add styleSrc and fontSrc arrays to tell it to allow Google. Update it to look like this:

TypeScript
const { nonce, header, csp } = createContentSecurityPolicy({
    shop: {
        checkoutDomain: context.env.PUBLIC_CHECKOUT_DOMAIN,
        storeDomain: context.env.PUBLIC_STORE_DOMAIN,
    },
    styleSrc: [
        "'self'",
        "'unsafe-inline'",
        "https://cdn.shopify.com",
        "https://fonts.googleapis.com", 
    ],
    fontSrc: [
        "'self'",
        "data:",
        "https://fonts.gstatic.com",
    ],
});
3. Restore the Missing Hero Image
The very last error in your log is 404 (Not Found) for /images/hero-image.jpeg.

In a standard React or Liquid app, you might just throw images anywhere. But in Hydrogen, the server only publicly serves files that are placed inside a specific folder called public.

Look at your Replit file tree. Do you have a folder named public?

If your image is currently sitting somewhere else (like in an assets folder inside app), you need to move it!

Create an images folder inside the public folder, and move hero-image.jpeg into it.

The final path should be public/images/hero-image.jpeg.
